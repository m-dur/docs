April 2026
Production hardening & brokerage detail
Production Fidelity integration, holdings redesign, reliability improvements & documentation restyle
  • Fidelity production integration: Upgraded SnapTrade from test to production credentials, connected Fidelity, and added daily sync of accounts, balances, holdings, and activities
  • 401(k) cost basis derivation: Reconstructs cost basis from contribution history for retirement accounts where brokers do not expose it directly, so P&L and total return display correctly
  • Holdings view variants: Two new view modes — an editorial Ledger view and a trading-desk Terminal view — alongside the original Classic table, with per-user preference persisted across sessions
  • Group-by modes: Holdings can now be grouped by brokerage, account, symbol, or flattened, with same-ticker positions across brokerages consolidated into a single aggregated row
  • Schwab token reliability fix: Corrected a refresh-token handling bug that silently shifted the database’s recorded expiry and caused intermittent integration failures around the 7-day mark
  • Brokerage expiry alerting: Proactive notifications before brokerage OAuth tokens expire, so re-authentication happens before syncing breaks
  • External liveness monitoring: Independent operational check that surfaces silent pipeline failures — stale snapshots, scheduler outages, database crash loops — even when the orchestrator itself is down
  • Net worth snapshot resilience: The daily snapshot now runs and records the latest-available data even when upstream brokerage syncs fail, preferring stale-but-present history over gaps
  • Per-symbol chart continuity: Stock history charts now merge activity from multiple sources and preserve transfer direction, so positions that moved between brokerages show one continuous line instead of dropping or doubling
  • Activity deduplication: Removed duplicate transaction records created during an earlier CSV import, fixing apparent position doubling in per-symbol charts
  • Residual holding reconstruction: Post-transfer residual fractional shares now show accurate historical values using current quantities and historical prices
  • Notification overhaul: Daily Telegram and Discord updates switched to the authoritative net worth source, added a per-stock gain/loss summary, and combined bank and investment breakdowns
  • Public landing page: Static marketing page on a custom domain to satisfy brokerage-partner review without exposing personal financial data
  • Documentation restyle: Changelog moved off the generic purple template to a warm editorial palette with distinctive typography
Key Code Changes
  • SnapTrade Production Service — Reusable module for production-client sync covering accounts, equity, holdings, activities, and cost-basis backfill
  • Stock Price Tracker DAG — Added Fidelity prod sync task, token-expiry alert task, and fixed the net-worth snapshot trigger rule so it runs even when upstream syncs fail
  • Financial Data DAG — Discord notifications expanded with net worth change and per-stock gain/loss table
  • Schwab Service — Refresh-token flow preserves initial-grant expiry instead of sliding it forward on every access-token rotation
  • SnapTrade Routes — Stock history endpoint merges multiple sources and respects transfer direction for accurate per-symbol charts
  • HoldingsLedgerView / HoldingsTerminalView — New React components with a shared grouping helper and collapsible institution sections
  • SnapTradePage.jsx — Holdings tab gained view-mode and group-by switchers with preference persistence
